# kk哥的个人博客 - 快速开始指南

## 🎯 目标

在你的电脑上克隆这个项目，修改代码，然后更新到阿里云服务器。

## 📋 步骤

### 第一步：在你的电脑上克隆仓库

```bash
# 1. 在GitHub/GitLab等平台创建一个新仓库
# 2. 克隆仓库到你的电脑
git clone <你的仓库地址> kk-website
cd kk-website
```

### 第二步：安装依赖并启动

```bash
# 安装依赖
npm install

# 启动服务器（推荐）
./run.sh

# 或者使用npm
npm start
```

### 第三步：访问网站

打开浏览器，访问：http://localhost:3000

### 第四步：修改代码

```bash
# 修改前端（HTML/CSS/JavaScript）
vim public/index.html

# 修改后端（Node.js/Express）
vim server.js

# 修改后保存文件，浏览器会自动刷新
```

### 第五步：提交修改

```bash
# 查看修改状态
git status

# 添加修改
git add .

# 提交修改
git commit -m "修改了导航栏样式"

# 推送到远程仓库
git push origin master
```

### 第六步：更新到阿里云服务器

```bash
# 在阿里云服务器上
cd /path/to/kk-website

# 拉取最新代码
git pull origin master

# 重启服务器
./deploy.sh

# 或者手动重启
pm2 restart kk-website
```

## 🔧 故障排除

### 1. 端口被占用
```bash
# 查看3000端口占用情况
lsof -i :3000

# 杀掉占用端口的进程
kill -9 <PID>

# 或者修改server.js中的PORT变量
```

### 2. 依赖安装失败
```bash
# 删除node_modules和package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 3. 服务器启动失败
```bash
# 查看详细错误
node server.js

# 或者查看日志
pm2 logs kk-website
```

### 4. Git推送失败
```bash
# 检查远程仓库地址
git remote -v

# 如果地址错误，修改远程地址
git remote set-url origin <正确的地址>

# 然后重新推送
git push origin master
```

## 📊 网站功能

### 1. 博客展示
- 个人简介
- 兴趣爱好
- 损友故事（需要解锁）

### 2. 留言功能

- 左侧边栏导航

## 🎯 开发建议

### 1. 小步提交
- 每次修改只做一个功能
- 提交信息要清晰
- 频繁提交，便于回滚

### 2. 测试验证
- 修改后先本地测试
- 确保功能正常再提交
- 使用测试脚本验证

### 3. 文档更新
- 修改功能后更新文档
- 记录重要的修改
- 保持文档同步

### 4. 备份数据
- 定期备份数据库
- 使用Git管理代码
- 保留重要版本
