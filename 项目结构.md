# kk哥的个人博客 - 项目结构说明

## 📁 项目目录结构

```
kk-website-git/
├── .git/                    # Git仓库目录
├── .gitignore              # Git忽略文件
├── README.md               # 项目说明文档
├── 使用说明.md             # 使用说明文档
├── 项目结构.md             # 本文件
├── package.json            # Node.js项目配置
├── package-lock.json       # 依赖锁定文件
├── server.js               # 服务器主文件
├── run.sh                  # 启动脚本（推荐）
├── start.sh                # 启动脚本（备用）
├── deploy.sh               # 部署脚本
├── test.sh                 # 测试脚本
├── guestbook.db            # SQLite数据库文件
└── public/                 # 前端文件目录
    ├── index.html          # 主页面
    ├── index.html.backup   # 备份文件
    └── index.html.backup2  # 备份文件
```

## 📄 文件说明

### 配置文件

- **.gitignore** - Git忽略文件，排除不需要的文件（如node_modules、日志文件等）
- **package.json** - Node.js项目配置文件，包含项目信息、依赖和脚本
- **package-lock.json** - 依赖锁定文件，确保依赖版本一致

### 文档文件

- **README.md** - 项目说明文档，包含项目概述、技术栈、运行方式等
- **使用说明.md** - 详细的使用说明，包含快速开始、开发指南、部署说明等
- **项目结构.md** - 本文件，说明项目目录结构和文件作用

### 服务器文件

- **server.js** - 服务器主文件，包含Express服务器、API接口、数据库连接等
- **run.sh** - 启动脚本（推荐使用），包含错误检查和详细输出
- **start.sh** - 启动脚本（备用），原始启动脚本
- **deploy.sh** - 部署脚本，用于在服务器上部署项目
- **test.sh** - 测试脚本，用于测试网站功能

### 数据库文件

- **guestbook.db** - SQLite数据库文件，存储留言板数据

### 前端文件

- **public/index.html** - 主页面文件，包含HTML、CSS、JavaScript
- **public/index.html.backup** - 备份文件（2026-03-01）
- **public/index.html.backup2** - 备份文件（2026-03-02）

## 🎯 核心文件详解

### server.js

服务器主文件，包含以下功能：

1. **Express服务器**
   - 静态文件服务
   - API接口
   - 错误处理

2. **SQLite数据库**
   - 数据库连接
   - 表结构创建
   - 数据操作

3. **API接口**
   - `/health` - 健康检查
   - `/api/about` - 个人详细信息
   - `/api/roasts` - 损友语录
   - `/api/friends` - 损友详细信息
   - `/api/comparison` - 损友对比
   - `/api/guestbook` - 留言板（GET/POST）
   - `/api/guestbook/stats` - 留言统计
   - `/api/guestbook/:id` - 删除留言

### public/index.html

前端主页面，包含：

1. **HTML结构**
   - 博客头部
   - 左侧边栏（导航）
   - 中间内容区域
   - 右侧边栏（预留）

2. **CSS样式**
   - 黑白配色
   - 极简高级风格
   - 响应式设计
   - 动画效果

3. **JavaScript功能**
   - 板块切换（留言板、关于我、好友专属）
   - 留言功能
   - 损友内容解锁
   - 导航栏激活状态

## 🔧 开发工作流

### 1. 本地开发

```bash
# 克隆仓库
git clone <your-repo-url> kk-website
cd kk-website

# 安装依赖
npm install

# 启动开发服务器
npm start

# 或者使用启动脚本
./run.sh
```

### 2. 修改代码

```bash
# 修改前端
vim public/index.html

# 修改后端
vim server.js

# 提交修改
git add .
git commit -m "描述你的修改"
git push origin master
```

### 3. 部署到服务器

```bash
# 在服务器上
cd /path/to/kk-website
git pull origin master
./deploy.sh
```

## 📝 开发建议

### 前端开发

1. **修改HTML**
   - 直接修改 `public/index.html`
   - 注意保持HTML结构完整
   - 测试浏览器兼容性

2. **修改CSS**
   - 在HTML的 `<style>` 标签中修改
   - 或者创建单独的 `style.css` 文件
   - 注意响应式设计

3. **修改JavaScript**
   - 在HTML的 `<script>` 标签中修改
   - 或者创建单独的 `script.js` 文件
   - 注意浏览器兼容性

### 后端开发

1. **修改API接口**
   - 在 `server.js` 中添加新的API接口
   - 注意错误处理
   - 注意安全性

2. **修改数据库操作**
   - 注意SQL注入防护
   - 注意数据验证
   - 注意错误处理

### 部署建议

1. **使用PM2**
   - 进程管理
   - 日志管理
   - 自动重启

2. **使用Nginx**
   - 反向代理
   - SSL证书
   - 负载均衡

3. **备份策略**
   - 定期备份数据库
   - 备份代码到Git
   - 备份配置文件
